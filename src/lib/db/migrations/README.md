# Migrations

We use Drizzle Kit to generate SQL migrations from `src/lib/db/schema.ts`.

## Workflow

```bash
# After editing schema.ts:
npm run db:generate    # writes a new <hash>_<name>.sql file here

# Apply pending migrations to the DB pointed at by DATABASE_URL:
npm run db:migrate

# Inspect the DB visually:
npm run db:studio
```

## Files in this folder

- `0000_extensions.sql` — hand-written. Enables pgvector and pgcrypto, must
  run before any drizzle-generated migration. The runner applies files in
  lexicographic order, so `0000_` runs first.
- `0001_<hash>_<name>.sql` and onward — generated by `drizzle-kit generate`.
- `meta/` — Drizzle's bookkeeping. Don't edit by hand.

## After the first generate

The first `npm run db:generate` will create `0000_<name>.sql` and renumber
this file. If that happens, rename the generated extension file back to a
prefix that sorts BEFORE the schema migration (e.g. `0000_aaa_extensions.sql`).
